  <action>
    Create `scripts/validate-realtime.ts` that validates Supabase Realtime is working. The script should:

    1. Import the Supabase client from `@/lib/supabase` (or create a standalone client using the env vars `EXPO_PUBLIC_SUPABASE_URL=https://iaqztbhkukgghomwnict.supabase.co` and `EXPO_PUBLIC_SUPABASE_ANON_KEY` from `.env`)
    2. Create a channel named `infra-validation`
    3. Subscribe to `postgres_changes` on `public.profiles` table for `*` events
    4. Log subscription status transitions
    5. When status reaches `SUBSCRIBED`, log "SUCCESS: Realtime subscription active"
    6. When status is `CHANNEL_ERROR`, log "FAILED: Realtime subscription error" and exit with code 1
    7. After successful subscription, clean up the channel with `supabase.removeChannel(channel)`
    8. Exit with code 0 on success

    Use the existing `@supabase/supabase-js` package (already installed, version 2.95.3). The script should be runnable via `npx tsx scripts/validate-realtime.ts`.

    The validation pattern:
    ```typescript
    import { createClient } from '@supabase/supabase-js'

    const supabase = createClient(
      process.env.EXPO_PUBLIC_SUPABASE_URL!,
      process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY!
    )

    const channel = supabase
      .channel('infra-validation')
      .on('postgres_changes', {
        event: '*',
        schema: 'public',
        table: 'profiles'
      }, (payload) => {
        console.log('Event received:', payload)
      })
      .subscribe((status) => {
        console.log('Subscription status:', status)
        if (status === 'SUBSCRIBED') {
          console.log('SUCCESS: Realtime subscription active')
          supabase.removeChannel(channel)
          process.exit(0)
        }
        if (status === 'CHANNEL_ERROR') {
          console.error('FAILED: Realtime subscription error')
          process.exit(1)
        }
      })

    // Timeout after 10 seconds
    setTimeout(() => {
      console.error('FAILED: Subscription timed out after 10s')
      supabase.removeChannel(channel)
      process.exit(1)
    }, 10000)
    ```

    Also ensure `tsx` is available by checking if it is in devDependencies; if not, the executor should install it: `npm install -D tsx`.

    Load environment variables using dotenv: add `import 'dotenv/config'` at the top (install `dotenv` as devDependency if not present: `npm install -D dotenv`).
