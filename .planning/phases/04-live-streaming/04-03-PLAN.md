<task type="auto">
  <name>Task 1: Create useLiveChat and useLiveViewers hooks</name>
  <files>hooks/useLiveChat.ts, hooks/useLiveViewers.ts</files>
  <action>
  1. Create `hooks/useLiveViewers.ts`:
     - Export `function useLiveViewers(liveId: string, userId: string): number`
     - Creates a Supabase Presence channel: `supabase.channel('live-viewers:${liveId}')`
     - On subscribe (status === 'SUBSCRIBED'), calls `channel.track({ user_id: userId, joined_at: new Date().toISOString() })`
     - On presence 'sync' event, reads `Object.keys(channel.presenceState()).length` and sets viewerCount state
     - Cleanup: calls `channel.untrack()` then `supabase.removeChannel(channel)`
     - Returns viewerCount (number)
     - Dependencies: [liveId, userId]

  2. Create `hooks/useLiveChat.ts`:
     - Define `ChatMessage` interface: `{ id: string, user_id: string, display_name: string, avatar_url: string | null, text: string, sent_at: string }`
     - Export `function useLiveChat(liveId: string): { messages: ChatMessage[], sendMessage: (msg: ChatMessage) => void }`
     - Creates a Supabase Broadcast channel: `supabase.channel('live-chat:${liveId}')`
     - Subscribes to `on('broadcast', { event: 'message' }, ({ payload }) => ...)` events
     - Appends new messages to state, keeping last 100 messages max: `prev.slice(-99).concat(payload)`
     - `sendMessage` calls `channel.send({ type: 'broadcast', event: 'message', payload: msg })`
     - Cleanup: `supabase.removeChannel(channel)`
     - Uses useRef for channel to avoid stale closures in sendMessage callback
  </action>

  2. Create `app/live/[id].tsx` — full-screen live stream viewer:

     **Screen flow:**
     a. Receives `id` param (lives table UUID) via `useLocalSearchParams<{ id: string }>()`
     b. On mount, fetches the live row from Supabase: `supabase.from('lives').select('*, host:profiles!author_id(*)').eq('id', id).single()`
     c. If live.status !== 'live', show "Stream ended" UI with back button
     d. Calls `fetchLiveKitToken(live.room_name, false)` — canPublish:false for viewer
     e. Renders LiveKitRoom once token is ready

     **Layout (full-screen, StatusBar hidden):**
     ```
     <LiveKitRoom
       serverUrl={serverUrl}
       token={token}
       connect={true}
       options={{ adaptiveStream: { pixelDensity: 'screen' } }}
       audio={false}
       video={false}
     >
       <ViewerContent live={live} />
     </LiveKitRoom>
     ```

     - IMPORTANT: Call `AudioSession.startAudioSession()` in useEffect on mount, `AudioSession.stopAudioSession()` on cleanup (required for iOS audio playback)

     **ViewerContent component:**
     - Uses `useTracks([Track.Source.Camera])` to find the host's video track (remote participant, NOT local)
     - Renders `VideoTrack` filling the entire screen (`StyleSheet.absoluteFill`)
     - If no track yet, show loading state with cover image as background

     **Overlay elements (absolute positioned over video):**

     a. **Top bar** (SafeArea top):
        - Back/close button (X icon, top-left)
        - Host info: avatar + display_name + "LIVE" badge (top-left, next to close)
        - Viewer count badge (top-right): pill with eye icon + count from `useLiveViewers(id, userId)`

     b. **Chat overlay** (bottom half, semi-transparent):
        - Uses `useLiveChat(id)` for messages
        - FlatList/ScrollView of messages, newest at bottom, auto-scrolls
        - Each message: `display_name: text` with subtle text shadow for readability
        - Keep max ~6 visible messages to avoid covering the whole video

     c. **Chat input** (bottom, above safe area):
        - TextInput with send button
        - On send: creates ChatMessage with `{ id: uuid, user_id, display_name, avatar_url, text, sent_at }` from auth store/profile
        - Calls `sendMessage(msg)` from useLiveChat
        - Keyboard avoiding behavior (KeyboardAvoidingView)

     **Design:** Dark overlays with glassmorphism elements. Text should have text shadow for readability over video. Match the existing LiveSection card aesthetic (gradient overlays, rounded badges).

     **Cleanup on unmount:** All hooks clean up their channels. LiveKitRoom disconnects automatically. AudioSession stopped.
