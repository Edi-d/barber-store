  <name>Task 1: Create useRealtimeLives hook</name>
  <files>hooks/useRealtimeLives.ts</files>
  <action>
  Create `hooks/useRealtimeLives.ts`:

  1. Export `function useRealtimeLives(): { lives: LiveWithHost[], loading: boolean }`

  2. **Initial fetch:** On mount, query lives table for active streams:
     ```
     supabase
       .from('lives')
       .select('*, host:profiles!author_id(id, username, display_name, avatar_url, role)')
       .eq('status', 'live')
       .order('viewers_count', { ascending: false })
       .limit(20)
     ```
     Set loading=true before, false after. Store results in state as `LiveWithHost[]`.

  3. **Realtime subscription:** Subscribe to postgres_changes on the lives table:
     ```
     supabase.channel('realtime-lives')
       .on('postgres_changes', {
         event: '*',
         schema: 'public',
         table: 'lives',
       }, (payload) => { ... })
       .subscribe()
     ```

     Handle events:
     - **INSERT** with status='live' or **UPDATE** where new.status='live': Fetch the full row with host join (need the profile data), add to state if not already present
     - **UPDATE** where new.status='ended': Remove from state by id
     - **UPDATE** where status stays 'live': Update viewers_count or other fields in state
     - For INSERT/UPDATE that need host data, do a quick single-row fetch: `supabase.from('lives').select('*, host:profiles!author_id(...)').eq('id', newId).single()`

  4. **Cleanup:** Remove channel on unmount via `supabase.removeChannel(channel)`

  5. Import types from `@/types/database` and supabase from `@/lib/supabase`
